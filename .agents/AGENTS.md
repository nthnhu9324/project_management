# Agent Rules

## Code Style and Philosophy

* **Cleanliness**: Write clean, lint-compliant code. Remove all redundant comments, dead code, and "trash." Only include comments that explain the "why" rather than the "what."
* **Structure**: Prioritize a modular mindset. Ensure every component is decoupled, highly cohesive, and reusable. Organize logic into small, single-responsibility functions or classes.
* **Clarity**: Ensure code flow is linear and intuitive. Avoid clever hacks or premature optimizations that sacrifice readability.
* **Documentation**: Provide concise, accurate docstrings that follow standard conventions. Focus on functionality, parameters, and return types rather than implementation details.
* **Philosophy**: Always favor explicit, readable solutions over obscure brevity. If an architectural decision increases maintenance overhead, simplify it immediately. The goal is to deliver perfect, professional-grade code that is a pleasure to maintain.
